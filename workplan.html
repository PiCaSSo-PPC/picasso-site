Work Plan

Detailed breakdown of project tasks and their objectives

This task will focus on collecting the following: 1. data requirements: identify which data fields need to be persistently stored); 2) data security requirements to ensure protection of sensitive data and availability of the whole set of data stored; and 3) functional requirements: define the tasks the platform needs to perform to satisfy user information needs.

2

Clinical data registry design and implementation

ðŸŽ¯ Objective

This task is twofold: 1) to design and implement the Clinical Data Repository (CDR), a secure data repository for storing clinical data related to Pediatric Palliative Care (PPC); and 2) to design a detailed architecture for the integrated digital platform for PPC.

ðŸ”¬ Task Description

This task focuses on: 1) database modeling and implementation applied to the CDR; 2) data security techniques to ensure the protection of PPC sensitive data; and 3) component architecture of a software system that meets the objectives of our integrated digital platform for PPC.

The CDR will be designed and implemented according to the data requirements specified by the HSM Pediatrics Department.

3

Implementation of the Integrated Digital Platform

ðŸŽ¯ Objective

This task has three primary objectives: 1) to implement the server-side logic of our integrated digital platform to meet the functional requirements gathered in Task 1; 2) to design and develop the Graphical User Interface (GUI) of our integrated digital platform accessible through a Web browser; and 3) to migrate PPC data previously collected in an Excel file by the HSM Pediatrics Department team specialized in CPP into the CDR constructed in Task 3.

ðŸ”¬ Task Description

The task involves three main steps: (i) server implementation: Develop the server-side logic of our integrated digital platform, encompassing application logic for data input, search, update, and analysis within the PPC CDR; (ii) GUI design and development: design and implement a user-friendly and functional GUI for the platform; (iii) sata migration: create and execute a data migration pipeline to transfer existing data from an Excel file into the PPC CDR.

The GUI of the integrated digital platform will be designed to meet the usability needs specified by PPC experts from HSM. The HSM Pediatrics Department will provide the Excel file containing PPC data about its patients, which will be used for data migration into the CDR.
